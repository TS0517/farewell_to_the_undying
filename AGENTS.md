## UI実装後の自己検収ルール

GodotのUI・戦闘画面・画像素材配置を実装した場合、完了報告の前に必ず以下を確認すること。

### 必須チェック

- 1920x1080画面からUIがはみ出していない
- 主要UIが背景・キャラ・敵・3x3グリッドを隠していない
- 戦闘画面では3x3グリッドが必ず見えている
- 味方最大4人のステータスが読める
- 敵情報が読める
- 下部UIが戦闘フィールドを潰していない
- TextureRect / Sprite2D / Control は原寸表示任せにせず、表示サイズを明示制御している
- UI素材は必ず縮小・フィット表示し、原寸で巨大表示しない
- 画像素材に白背景・市松模様・不要な余白が焼き込まれていないか確認する
- 指定素材が存在しない場合、勝手に似た名前の画像を複製して使わない
- 指定素材がない場合は仮Panel / ColorRectで代用し、「素材未配置」と報告する
- 既存のUI・3x3グリッド・キャラ表示・敵表示を壊していない
- 変更前に存在していた重要表示が消えていない

## 戦闘画面UIの基準

- 敵情報は右上にコンパクト表示
- 味方4人ステータスは画面下部に横並び
- 下部UI高さは原則170〜220px以内
- 3x3グリッドは戦闘フィールド上に見える状態を維持
- 3x3グリッドと下部ステータスUIを重ねない
- UI素材サイズが大きい場合でも、Control側で必ず表示領域内に収める
- 左上に味方単体情報を大きく出さない。味方情報は原則として下部4人UIに集約する

## 禁止事項

- 指定素材がないのに、似た名前の画像を勝手に複製して指定名にすること
- 画像を原寸表示して画面を覆うこと
- UIが戦闘フィールドの半分以上を潰した状態で完了報告すること
- 3x3グリッドが見えない状態で完了報告すること
- 既存表示が消えたのに報告しないこと
- 「ヘッドレス読み込み確認が通った」だけで見た目OKと判断すること

## 完了報告に必ず含める内容

UI・戦闘画面・画像素材配置を変更した場合、完了報告には必ず以下を含めること。

1. 1920x1080での表示確認結果
2. 3x3グリッドが見えているか
3. 下部UIの実サイズ、またはおおよその高さ
4. 使用した画像素材名
5. 指定素材が無かった場合の代用内容
6. 既存表示を壊していないか
7. 残っている見た目上の懸念

## 合格できない場合

上記の必須チェックを1つでも満たせない場合は、完了報告せず、先に修正すること。
不明な場合は「見た目確認が未完了」と明記し、勝手に完了扱いしないこと。

## Git自動保存ルール

Codexは、このプロジェクト内でファイル変更を行った場合、ユーザーの追加指示がなくても作業完了時に必ずGit保存する。

作業完了時に必ず行うこと:
1. git status で変更内容を確認
2. git add .
3. 内容が分かる短いメッセージで git commit
4. git push origin main

commit message形式:
- add: ○○追加
- fix: ○○修正
- tweak: ○○調整
- chore: ○○整理

禁止:
- .godot/
- export/
- exports/
- 一時スクリーンショット
- battle_ui_capture.png
- OS一時ファイル
をcommitしないこと。

注意:
- 変更がない場合はcommitしない
- pushに失敗した場合は理由を報告する
- バグが後から見つかった場合は、Git履歴から前の状態へ戻す前提でよい
