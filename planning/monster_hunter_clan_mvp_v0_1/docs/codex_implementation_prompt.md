# Codex実装指示プロンプト

あなたはGodot/GDScriptのゲーム開発エージェントです。
以下の仕様に従って、スマホ横画面向けの10年版MVPを段階的に実装してください。

## 最重要方針
- まず動く骨格を作る。
- UIや画像は仮でよい。
- テキストはコード直書き禁止。localization IDを使う。
- キャラ、職業、スキル、プロフィール、会話はJSONから読み込む。
- 内部ロジックはシンプルにし、後からデータ追加で拡張できるようにする。

## 使用技術
- Godot
- GDScript
- 横画面 1920x1080 基準
- Android実機テストを想定

## 実装フェーズ

### Phase 1: プロジェクト基盤
- Godotプロジェクト作成
- 1920x1080横画面設定
- 入力はマウス/タップ両対応
- SceneManager相当の画面遷移基盤
- DataManagerでJSON読み込み
- LocalizationManagerでテキストID解決

### Phase 2: データ読み込み
以下を読み込めるようにする。

- jobs.json
- skills.json
- profiles.json
- personas.json
- portraits.json
- dialogues.json
- monsters.json
- localization_ja.json

### Phase 3: 酒場画面
酒場候補キャラをカード表示する。

表示内容:
- 顔画像
- 名前
- 職業
- 現年齢
- 全盛期
- 潜在能力
- STR/AGI/VIT/INT
- 前列/中列/後列スキル
- 人格タグ
- プロフィール
- 雇用ボタン

雇用するとパーティ保有リストに追加。
最大8人。

### Phase 4: 王国画面
表示:
- 現在年
- 季節
- 討伐数
- 王国安定度
- つぶやき
- メニュー: 酒場、訓練、引退、墓、戦闘へ

### Phase 5: 戦闘画面
- 3x3グリッド
- 味方最大4人
- 敵1体
- ATB自動行動
- キャラタップ -> 移動先タップで移動
- 移動時ATBリセット
- HP0で仲間は戦死、王子は気絶
- 勝利/敗北判定

### Phase 6: 年齢と成長
1年終了時に全員1歳加齢。

年齢処理:
- prime_start未満: 成長
- prime_start〜prime_end: 維持
- prime_end超過: 衰退

成長に影響:
- 潜在能力
- 戦闘参加回数
- とどめ回数

### Phase 7: 戦死/クリスタル/引退/病死
- 戦死した仲間はクリスタル化
- クリスタルは王子の装備枠2つに装備可能
- MVPではステータス加算のみ
- 引退者は役割を選ぶ: 教師/衛兵/商人・鍛冶
- 衰退期以降は冬終了時に低確率で病死判定

### Phase 8: 会話
トリガーに応じてdialogues.jsonからセリフを選ぶ。

必要なフィルタ:
- trigger
- persona_id
- age_stage
- priority
- used_countまたはrecent history

表示場所:
- 戦闘リザルト
- 王国つぶやき
- 手紙
- 酒場自己紹介

## 注意
- UIはスマホ横画面で押しやすくする。
- まず日本語だけでよいが、多言語構造は最初から入れる。
- スキルクリスタル、武器屋完全実装、10職業、50年版は後回し。
